一、拖拽

1. 拖拽概念

鼠标移动到某个元素上，按下鼠标并拖动，元素会跟随鼠标的移动而移动。例如：图形验证码的滑块，网页右侧滚动条等等。

2. 拖拽原理

  • 点击元素拿到元素的初始位置和鼠标的初始位置

  • 移动的时候拿到鼠标最后的位置，可以求出鼠标的距离差

  • 元素最终的位置 = 元素的初始位置 + 鼠标的距离差

3. 案例

3.1 拖动小盒子移动

拖动事件可能出现的问题：

1、move事件在鼠标Up之后不会自动消失。

解决办法：在up事件当中需要将鼠标的move事件和up事件一起取消掉。

2、快速拖动的时候会有鼠标移动盒子没跟着移动的情况。

问题分析：这是由于鼠标移动由操作系统控制，而盒子的位置需要计算，会慢一些，当鼠标移动过快超出盒子位置的时候，盒子的mousemove事件不会被触发，导致盒子没有跟随鼠标而移动。 解决办法：把mousemove和mouseup事件添加在document上，使用事件冒泡去解决元素的移动。

3、当盒子里有文字，选中文字的时候，出现了拖动不了盒子的问题。

问题分析：当在元素内部我们有文本，文本可以拖拽，默认处理了这个事件，所以要取消浏览器的默认行为。

解决办法：我们只需要阻止系统默认事件就可以，在mousemove（还能选中文本）/ mousedown（不能选中文本）事件里的最后return false或者e.preventDefault()阻止系统默认对事件的处理。

4、在ie8及8以下的版本，选中文字的时候拖动还有问题，会出现禁止拖动的图标。

问题分析：ie的默认行为不包含这个拖拽的，所以我们要进行全局捕获。

解决办法：在mousemove事件之前添加box.setCapture()，当鼠标点击元素之后，把鼠标的后序事件全部捕获到该元素身上；在mouseup事件里添加box.releaseCapture()，鼠标的事件全局捕获，有添加就要有释放，否则后序点击任何地方都相当于在点击box。由于高版本的浏览器没有setCapture方法，直接写会报错，所以要加判断。最终写法：box.setCapture && box.setCapture(); 和box.releaseCapture && box.releaseCapture()；。

3.2 自定义滚动条

1、我们先实现一个简单的、固定大小的，能在固定的滚动槽里滑动的滑块。

我们的页面架构首先要清楚，页面的最外层是document，紧接着是初始包含块 html body，其次是我们的元素，我们禁止系统的滚动条，（因为各大浏览器的系统滚动条风格不一，有可能会影响我们的页面布局），所以在body当中我们一般会有最外的一个盒子模拟body区域，在这个盒子的右侧会定位一个盒子模拟滚动条。

2、给网页上写入内容、根据内容设置滑块的高度、让网页内容跟着滑块滚动。

  • 给网页上写入内容：拿到添加内容的盒子，给innerHTML属性设置值就好。

  • 我们让滑块的高度不要固定，跟随内容区的高度来设置，根据自定义滚动条的万能比例：

    滑动块的高度 / 滑动槽的高度 = 屏幕的高度 / 内容的高度 = 滚动条的滚动距离 / 内容的滚动距离，

    我们可以计算出滑动块的高度并设置。

  • 让网页内容跟着滑块滚动，和设置滑动块的属性让滑动块向下滑动的思路一样，可以设置内容的相关属性，只是在滑块向下滑动的时候，内容区是向上移动的。

二、滚动事件

1. 概念

如果我们希望滚动鼠标的同时滚动条也跟着滚动或者实现滚动鼠标图片的放大/缩小功能的时候，我们可以监听鼠标的滚动事件，不同的浏览器监听的方法不一样。

  • ie / chrome：mousewheel，通过event.wheelDelta得到滚动一小格的默认值，鼠标向上滚动值为120，鼠标向下滚动为-120。
  • firefox：DOMMouseScroll，通过event.detail得到滚动一小格的默认值，鼠标向上滚动值为-3，鼠标向下滚动为3，需要注意的是，这个事件只能通过addEventListener注册。

2. 案例

网页中间放一个盒子，当鼠标滚轮向下时，盒子高度逐渐变大；鼠标滚轮向上时，盒子高度逐渐减小的效果。

三、轮播图

1. 案例

轮播图也称为焦点图，是网页中比较常见的网页特效，这一部分我们就要实现一个轮播图，效果图如下：

布局不需要我们实现，会直接提供给大家，这里我们主要实现js部分的代码，我们先来整理一下需求。

功能需求：

​ 1. 鼠标经过轮播图模块，左右按钮显示，离开隐藏左右按钮。

​ 2. 点击一次左边箭头，图片往左播放一张；点击一次右边箭头，图片往右播放一张。

​ 3. 添加图片的无限循环，总共有五张图，但需要这五张图能一直轮流朝一个方向切换。

​ 4. 图片切换的同时，当前显示的图片对应的小圆点变红色。

​ 5. 点击对应位置的小圆点，可以实现图片切换。

​ 6. 鼠标不经过轮播图时，轮播图自动切换图片。

​ 7. 鼠标经过，轮播图停止自动切换图片。

2. 节流阀

防止轮播图按钮连续点击造成播放过快。

节流阀目的：当上一个函数动画内容执行完毕，再去执行下一个函数动画，让事件无法连续触发。

核心实现思路：利用回调函数，添加一个变量来控制，锁住函数和解锁函数。

开始设置一个变量var flag= true;

if(flag){flag = false; do something}，关闭水龙头。

利用回调函数动画执行完毕， flag = true，打开水龙头。
