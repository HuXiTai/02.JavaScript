1. 构造函数的理解

  • 通过Object创建的对象类别不确切，我们需要自己能创建出确切的类别对象，就需要自定义构造函数，自定义构造函数通常简称构造函数。

  • 构造函数本质上还是一个函数，任何的函数都可以当普通函数用也可以当构造函数用，就看我们调用函数的时候有没有加new，加了就是把函数当构造函数用，没加还是当普通函数用。

  • window代表浏览器窗口对象，所有的东西都是归window去管的，它是我们的顶级对象， 全局变量及函数最终都是会作为window对象的属性来对待,只不过一般window是省略不写的；

    function Car(name,color,price) {
    this.name = name;
    this.color = color;
    this.price = price;
    this.run = function() {
        console.log('跑的快~');
    }

    var a = 100;// 局部变量就是一句普通的功能代码，代表定义了一个变量，它既不是window的属性，也不是函数对象的属性
    // 只有全局变量特殊对待  会作为window对象的属性
    // this.a = a;
    // window.a = a;
    // return [1,2,3];
}

var result = Car('奔驰','black',200000);//此时就说这个函数是当普通函数用的
console.log(result);
//此时这个函数内部的this是window
//函数的返回值看的就是return,没有return就是undefined;
console.log(color);//普通函数调用，本质是在window对象上增加了name color price run属性和值
//在外部，我们就可以通过这个window的属性拿到函数内部的局部变量的值；

var c1 = new Car('劳斯莱斯','red',10000000);//此时就说这个函数是当构造函数用的
//此时这个函数内部的this 是准备实例化出来的对象，因为构造函数本身就是实例化对象用的；
//返回值：如果没写return或者return后面是基本值，那么返回的就是实例化出来的对象本身，和return没关系
// 如果return后面是对象类型的数据，那么返回的就是对象数据，和实例化出来的对象没关系
console.log(c1);
c1.run();

4.1 构造函数的问题

4.2 构造函数原型对象

概念：

JavaScript 规定，每一个构造函数都有一个prototype 属性，这个prototype属性指向另一个对象，这个对象的所有属性和方法，都会被构造函数所拥有，那么这个对象就叫原型对象。

特点：

伴随函数的定义而出现的另外一个对象。

默认是Object的实例对象。

function Star(uname, age) {
    this.uname = uname;
    this.age = age;
}
Star.prototype.sing = function() {
    console.log('我会唱歌');
}
var ldh = new Star('刘德华', 18);
var zxy = new Star('张学友', 19);
ldh.sing();//我会唱歌
zxy.sing();//我会唱歌

4.3 原型对象分类

显式原型对象：函数对象身上的prototype属性值，通常通过它来写这个原型对象。

隐式原型对象：实例化对象身上的__proto__属性值，通常通过它来读这个原型对象。

对象都会有一个属性 __proto__ 指向构造函数的 prototype 原型对象，之所以我们对象可以使用构造函数 prototype 原型对象的属性和方法，就是因为对象有 __proto__ 原型的存在。
__proto__对象原型和原型对象 prototype 是等价的。
__proto__对象原型的意义就在于为对象的查找机制提供一个方向，或者说一条路线，但是它是一个非标准属性，因此实际开发中，不可以使用这个属性，它只是内部指向原型对象 prototype。

4.4 原型对象的作用

资源共享 节约内存

4.5 constructor构造函数

对象原型 __proto__和构造函数prototype原型对象里面都有一个属性 constructor 属性 ，用于记录该对象引用于哪个构造函数，它可以指回构造函数本身。

4.5 原型链

每一个实例对象都有一个__proto__属性，指向构造函数的原型对象，构造函数的原型对象也是一个对象，也有__proto__属性，这样一层一层往上找就形成了一条链，我们称为原型链。

当访问一个对象的属性（包括方法）时，首先查找这个对象自身有没有该属性。 如果没有就查找它的原型（也就是__proto__指向的 prototype 原型对象）。 如果还没有就查找原型对象的原型对象（Object的原型对象）。 依此类推一直找到null为止。

4.6 原型链的作用

原型链的作用：实现继承的功能

<script type="text/javascript">
    // 需求：Star的实例对象可以调用Person的方法；而Person的实例对象不能调用Star的方法.
    function Person(name){
           this.name = name;
    }
    Person.prototype.eat =function(){
        console.log("---eat---");
    } 

    function Star(name, age) {
        this.name = name;
        this.age = age;
    }
    // 给Star设置原型对象为Person的原型对象，导致Person的实例对象也能调用Star的原型对象的方法。
    // Star.prototype = Person.prototype;
    Star.prototype = new Person();
    Star.prototype.constructor = Star;

    Star.prototype.dance = function() {
        console.log("---" + this.name + " dancing---");
    }

    var zxy = new Star("zxy",48);
    console.log(zxy.__proto__);
    console.log(Star.prototype);
    zxy.dance();
    zxy.eat();

    var person = new Person("gfc");
    person.dance();
</script>

4.7 原型对象中的this指向

构造函数中的this和原型对象的this，都指向我们new出来的实例对象。

function Star(uname, age) {
    this.uname = uname;
    this.age = age;
}
var that;
Star.prototype.sing = function() {
    console.log('我会唱歌');
    that = this;
}
var ldh = new Star('刘德华', 18);
// 1. 在构造函数中,里面this指向的是对象实例 ldh
console.log(that === ldh);//true
// 2.原型对象函数里面的this 指向的是 实例对象 ldh

案例：通过原型为数组扩展内置方法

Array.prototype.sum = function() {
    var sum = 0;
    for (var i = 0; i < this.length; i++) {
        sum += this[i];
    }
    return sum;
};
var arr = [1, 2, 3, 4, 5];
console.log(arr.sum());
//此时数组对象中已经存在sum()方法了  可以始终 数组.sum()进行数据的求和

5. 改变函数内部 this 指向

5.1 call方法

call()方法调用一个对象。简单理解为调用函数的方式，但是它可以改变函数的 this 指向

应用场景: 经常做继承. 

var o = {
    name: 'andy'
}
function fn(a, b) {
    console.log(this);
    console.log(a + b);
};
fn(1, 2)// 此时的this指向的是window 运行结果为3
fn.call(o, 1, 2)//此时的this指向的是对象o,参数使用逗号隔开,运行结果为3

5.2 apply方法

apply() 方法调用一个函数。简单理解为调用函数的方式，但是它可以改变函数的 this 指向。

应用场景: 经常跟数组有关系

var o = {
    name: 'andy'
}
function fn(a, b) {
      console.log(this);
      console.log(a + b)
};
fn(1, 2)// 此时的this指向的是window 运行结果为3
fn.apply(o, [1,2])//此时的this指向的是对象o,参数使用数组传递 运行结果为3

5.3 bind方法

bind方法也可以改变函数内部this的指向，但是它是通过返回一个改变了this指向的新的函数，我们通过调用这个新的函数来达到执行原函数并改变this指向的目的。

var o = {
    name: 'andy'
}
function fn(a, b) {
      console.log(this);
      console.log(a + b)
};
fn()// 此时的this指向的是window 运行结果为3
var new_fn = fn.bind(o);
new_fn(o, 1, 2)//此时的this指向的是对象o,参数使用数组传递 运行结果为3

5.4 call、apply、bind三者的异同

  • 共同点: call、apply、bind会调用函数, 并且改变函数内部this指向
  • 不同点: call 和 apply传递的参数不一样, call传递参数使用逗号隔开, apply使用数组传递；bind会生成新的函数，参数与call传递参数的方式一样。

5.5 改变函数内部this的应用

1、改变函数内部的this指向以帮助其他函数初始化属性

function Father(name, age, sex) {
    this.name = name;
    this.age = age;
    this.sex = sex;
}

function Son(name, age, sex, score) {
    this.score = score;
    // 调用Father函数，通过改变Father函数内部的this指向来为Son添加属性。
    // 此时call函数里的this指向的是Son的实例对象。
    Father.call(this, name, age, sex);
}

var father = new Father("qiangdie",48,"man");
var son = new Son("xiaoqqiang", 18, "man", 600);
console.log(father);
console.log(son);

2、让当前的对象去掉其他构造函数里的方法

var xf = {
    name: "xiaofang"
}

function Dog(name, age) {
    this.name = name;
    this.age = age;
}
Dog.prototype.eat = function() {
    console.log(this.name + "吃肉肉~~~");
}

var taidi = new Dog("taidi", 5);
// taidi.eat();

taidi.eat.call(xf);

6.2 instanceof

instanceof 用来判断一个对象是哪个构造函数的实例用的，专门用来解决判定数组和对象的时候使用。

// a = {};
// a = [1,2,3];
// a = function(){};
a = null;

console.log(null instanceof Object);//null并不具备任何对象的特性
console.log(a instanceof Array);// 使用这个方法可以判定它是不是一个数组
console.log(a instanceof Object);// 任何的对象都是Object的实例
console.log(window instanceof Object); // window也是一个对象
