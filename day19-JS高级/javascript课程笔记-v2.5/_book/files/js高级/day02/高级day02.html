二、面向对象特性

面向对象有三大特性：继承、封装、多态。

2.1 继承

js里继承分为：原型继承、构造函数继承、组合继承。

  • 原型继承：让父类的实例作为子类的原型，将子类的原型构造器补充完整 （为了让子类继承方法）
  • 构造函数继承：在子类当中去调用父类的构造函数（为了让子类继承属性）
    • 注意：子类可以从父类那里继承，父类却不能从子类继承。
  • 组合继承：原型继承方法，借用构造函数继承属性一起使用。
// 借用父构造函数继承属性
// 1. 父构造函数
function Father(uname, age) {
    // this 指向父构造函数的对象实例
    this.uname = uname;
    this.age = age;
    console.log(this)
}
Father.prototype.money = function() {
    console.log(100000);
};

// 2. 子构造函数 
function Son(uname, age, score) {
    // this 指向子构造函数的对象实例
    // this.uname = uname;
    // this.age = age;
    this.score = score;
    //借助父类的构造函数实现属性的继承
    Father.call(this, uname, age);
}
// 这样直接赋值会有问题,如果修改了子原型对象,父原型对象也会跟着一起变化
// Son.prototype = Father.prototype; 

//为了让子类去继承父类原型当中的方法
//我们需要用到原型继承，原型继承主要就是为了让子类继承方法使用的
//让子类的原型变成父类的一个实例
Son.prototype = new Father();
// 如果利用对象的形式修改了原型对象,利用constructor 指回原来的构造函数
Son.prototype.constructor = Son;
// 这个是子构造函数专门的方法
Son.prototype.exam = function() {
    console.log('孩子要考试');
}

2.2 多态

概念：不同的对象调用相同的方法产生不同的结果。

表现形式：方法的重载和重写。

  • 重载

    • 概念：用同一个函数完成不同的功能

    • 条件：(了解即可)

      1、函数名必须相同；

      2、函数参数必须不相同(可以是参数类型或者参数个数不同)；

      3、函数返回值可以相同，也可以不相同。

    • 作用：对于功能相同，仅仅是传参不同的函数，重载可以避免定义太多功能相同的函数。

    • 说明：js中的没有真正意义上的函数重载，因为 JS 中同一作用域下的同名函数，前者会被后者覆盖，但是可通过其他方法间接实现重载同样的效果。
    function Father(uname, age) {
    this.uname = uname;
    this.age = age;
}
Father.prototype.eat = function() {
    console.log("eat");
};
Father.prototype.eat = function(num) {
    console.log("eat" + num);
};
var father = new Father("zs", 18);
father.eat();
father.eat(8);

    间接实现重载的功能：

    function Father(uname, age) {
    this.uname = uname;
    this.age = age;
}
Father.prototype.eat = function() {
    if (arguments.length === 0) {
        console.log("eat");
    } else if (arguments.length === 1) {
        console.log("eat" + arguments[0]);
    }
};
var father = new Father("zs", 18);
father.eat();
father.eat(8);

  • 重写

    • 概念：和父类方法同名功能却不同。
    function Father(uname, age) {
    this.uname = uname;
    this.age = age;
}
Father.prototype.eat = function() {
    console.log("father eat");
}

function Son(uname, age, score) {
    this.score = score;
    Father.call(this, uname, age);
}
// 必须写在eat方法定义的前面
Son.prototype = new Father();
Son.prototype.constructor = Son;

Son.prototype.eat = function() {
    console.log("son eat");
}

function eat(man){
    // 不管传进来的是啥，调用它对应的eat方法。
   if (man.eat instanceof Function) {
       man.eat();
   }
}

var father = new Father("zs", 40);
var son = new Son("zw", 18, 100);

eat(father);
eat(son);

三、进程与线程

3.1 进程线程

  • 进程：系统进行资源分配和调度的一个独立单位，占有一片独有的内存空间。

  • 线程： CPU调度的基本单位, 没有单独的地址空间。

  • 区别：

    • 一个进程中至少有一个运行的线程: 主线程
    • 一个进程中也可以同时运行多个线程, 我们会说程序是多线程运行的
    • 一个进程内的数据可以供其中的多个线程直接共享
    • 多个进程之间的数据是不能直接共享的

3.2 单线程与多线程比较

  • 单线程
    • 优点：没有额外的开销
    • 缺点：效率低
  • 多线程
    • 优点：能有效提示cpu的利用率
    • 缺点：1、创建多线程需消耗资源；2、线程间切换需消耗资源；3、死锁与状态同步问题。

3.3 js是单线程

  1. 如何证明js执行是单线程的?

    答：定时器回调函数只有在运行栈中的代码全部执行完后才有可能执行（setTimeout的计时器不准的问题）。

  2. 为什么js要用单线程模式, 而不用多线程模式?

    答：JavaScript的单线程，与它的用途有关，作为浏览器脚本语言，JavaScript的主要用途是与用户互动，以及操作DOM。这决定了它只能是单线程，否则会带来很复杂的同步问题。

    虽然JS运行在浏览器中，是单线程的，但浏览器是多线程的。浏览器中很多异步行为都是由浏览器新开一个线程去完成。javascript引擎线程是浏览器多个线程中的一个，它本身是单线程的。浏览器还包括很多其他线程，如界面渲染线程，浏览器事件触发线程，Http请求线程等。

    所以，所谓的javascript是单线程，是指javascript引擎线程运行在浏览器中是单线程的。

3.4 模拟多线程（了解）

H5规范提供了js子线程的实现，叫Web Workers，每个线程可以向不同线程发送消息 也可以接收不同线程传来的消息。

相关API：

  • Worker: 构造函数, 加载分线程执行的js文件
  • Worker.prototype.onmessage: 用于接收另一个线程的回调函数
  • Worker.prototype.postMessage: 向另一个线程发送消息

主线程操作

  • 发送消息：worker.postMessage(消息可以是任何数据)

  • 接受消息： 

    worker.onmessage = function(e){
    console.log(e.data);// 接收到的消息或者数据在事件对象的data属性当中
}

子线程操作

  • 发送消息：this.postMessage(消息可以是任何数据)

  • 接受消息： 

    this.onmessage = function(e){
    console.log(e.data);// 接收到的消息或者数据在事件对象的data属性当中
}

不足：

  • worker内代码不能操作DOM(更新UI)

  • 不能跨域加载JS

            * 不是每个浏览器都支持这个新特性

案例：

计算得到fibonacci数列中第n个数的值 在主线程计算: 当位数较大时, 会阻塞主线程, 导致界面卡死 在子线程计算: 不会阻塞主线程

主线程：

var worker = new Worker('index.js');
worker.postMessage(40);
worker.onmessage = function(e) {
    console.log(e.data);
};
console.log("---");

子线程：

function fibo(n) {
    if (n < 2) {
        return n;
    }
    return fibo(n - 1) + fibo(n - 2);
}

this.onmessage = function(e) {
    var res = fibo(e.data);
    this.postMessage(res);
}

四、深浅拷贝

深拷贝和浅拷贝都是针对较为复杂的Object类型的数据，对普通的基本数据类型，变量名和值都存储在栈内存中，不会考虑它们的深浅拷贝。

4.1 浅拷贝

拷贝后的数据不一样，但对象里属性的值是同一个。

var obj = {
    name: "laoyang",
    do: function () {

    },
    score: [10, 20, 30]
}

//这不叫复制 而是给obj又起了一个名字
// var obj2 = obj;

var obj2 = {
    score: 0x111
};
//遍历obj 把obj的属性和值一个个拿过来
for (var key in obj) {
    obj2[key] = obj[key];
}

console.log(obj2);
console.log(obj2 === obj);

obj2.score.pop();
console.log(obj.do === obj2.do);

console.log(obj.score);

4.2 深拷贝

拷贝后的数据不一样，数据里属性的值也不一样。

var obj = {
    name: "laowang",
    do: function () {

    },
    score: [10, 20, 30]
}

function checkType(o) {
    return Object.prototype.toString.call(o).slice(8, -1).toLowerCase();
}

function deepClone(o) {
    //先判断o的类型是什么
    if (checkType(o) === "object") {
        //如果拷贝的是对象，则创建一个新对象
        var newObj = {};
    } else if (checkType(o) === "array") {
        //如果拷贝的是数组，则创建一个新数组
        var newObj = [];
    } else {
        // 如果是其他类型，则可以直接赋值，不需要拷贝
        return o;
    }

    //开始拷贝
    for (var key in o) {
        newObj[key] = deepClone(o[key])
    }

    return newObj;

}

var re = deepClone(obj);
console.log(re);
console.log(re.score === obj.score);

4.3 区别

如何区分深拷贝与浅拷贝？

1、假设B复制了A，当修改A时，看B是否会发生变化，如果B也跟着变了，说明这是浅拷贝，如果B没变，那就是深拷贝。

2、浅拷贝的for...in只循环第一层；而深拷贝是采用递归去拷贝所有层级属性。
